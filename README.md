🏠 LiVi Unity商城，我叫它团结改造家

> 初学者入门 Spring Boot 练手项目 😊 功能仍在迭代打磨，欢迎各位前辈、小伙伴提建议、指正问题~

一、项目介绍 📖

这是一个前后端分离的后端服务项目，基于 Spring Boot 框架开发，核心围绕三大生活化场景搭建：

- 家具商城 🛋️：实现基础商品展示、购物车管理、简单下单等电商核心逻辑；

- 智能家居控制 🏡：聚焦模拟设备连接、基础指令下发（比如灯光开关、空调温度调节），暂未对接真实硬件；

- 社交互动 🗣️：支持用户关注、简单动态发布功能，主打家具使用体验分享与智能生活经验交流。

当前项目处于初期开发阶段，代码存在不少可优化空间，后续会持续迭代完善功能~

二、核心功能（持续开发中...）📋

1. 主打卖家具电器的电商平台，该有的功能都想有，但现在还没有~

2. 智能家具控制，最后实际就控制了个字段状态(device_status)

3. 登录/注册,邮箱验证码...

4. WebSocket的一对多聊天群，边学边开发的，基本实现。

5. 未完待续...

三、技术栈选型 🛠️

（初学者友好型技术组合，以实用易上手为核心原则）

- 核心框架：Spring Boot

- 数据库：MySQL 8.0

- ORM 框架：MyBatis-Plus（简化CRUD操作，降低入门难度）

- 开发工具：IntelliJ IDEA + Git

四、本地运行步骤 🚀

1. 克隆仓库到本地：git clone [仓库地址]（将 [仓库地址] 替换为实际仓库链接）

2. 打开 IntelliJ IDEA，导入项目，等待 Maven 自动下载完成依赖 📥

3. 修改MySQL数据库配置

4. 找到 src/main/resources/application.yml 文件，修改数据库连接配置（替换为自己的MySQL用户名、密码、数据库名）

5. 定位到项目启动类（xxxApplication.java），右键运行启动 Spring Boot 项目


五、项目结构 📂

src/main/java/com/anxu/livi/
├── common/       // 通用工具类、配置类
│   ├── annotation/ // 自定义注解
│   ├── aspect/   // 自定义切面
│   ├── enums/    // 枚举类
│   ├── exception/ // 全局异常处理器
│   └── interceptor/    // Spring拦截器，对Http请求和WS握手进行安全校验
├── conf/         // 配置类
├── controller/   // 接口控制器
├── service/      // 业务逻辑层
├── mapper/       // Mapper接口（数据库操作相关）
├── model/        // 实体类、DTO、VO封装
│   ├── entity/   // 数据库表对应实体
│   ├── dto/      // 入参数据传输对象
│   └── vo/       // 出参数据封装对象
├── schedule/     // Spring轮询任务
├── util/         // 工具类
└── [项目名]Application.java  // 项目启动类
src/main/resources/
├── application.yml  // 全局配置文件（数据库、端口等）
└── com/anxu/livi.mapper/    // Mapper XML 映射文件目录


六、注意事项 ⚠️

- 本项目仅用于学习练手，未针对生产环境做优化（如缓存、分布式部署等），请勿直接用于生产场景；

- 智能家居控制功能为纯模拟实现，无真实硬件交互，拟打算使用Unity3D模拟智能设备的交互界面；

- 由于本人是初学者，代码中可能存在不规范、不合理之处，非常欢迎各位前辈提 PR 或 issue 指正 🙏；

- 开发过程中若遇到问题，可参考 Spring Boot 官方文档，也可以在仓库留言交流。
